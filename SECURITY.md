# Security Policy

## Supported Versions

Training Wheels is still pre-1.0. Security fixes are applied on a best-effort
basis to:

- the latest commit on `main`
- the most recent tagged release, once releases are published

Older snapshots and untagged historical commits are not supported.

## Reporting a Vulnerability

Do not open public GitHub issues for suspected vulnerabilities.

Report vulnerabilities privately by emailing `thgrace12@gmail.com` with:

- a clear description of the issue
- steps to reproduce or a proof of concept
- the affected version, commit, or deployment context
- any suggested mitigations, if known

If GitHub private vulnerability reporting is enabled for this repository, you
may use that channel instead.

## Response Expectations

- Initial acknowledgment target: within 5 business days
- Status updates: as new information is available
- Public disclosure: after a fix or mitigation is available, when practical

Please avoid sharing exploit details publicly until maintainers have had a
reasonable chance to investigate and respond.
